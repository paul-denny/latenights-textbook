# Binary Search
Binary search is a searching method that only works on _sorted_ data.
It uses a divide and conquer method to continually halve the values we need to search in a list.
Because of this, binary search is very efficient

Binary search works by selecting the middle value in a sequence and comparing that value to a target value.
If the value is equal to the target, we can stop.
Otherwise, if the target value is greater than the middle of the list we know, due to the sorted nature of the list, that there is no way the target value will be in the lower half of the list.
Therefore, in just one step we have eliminated half of the values from consideration in our search.
We can then repeat that process on the remaining half of the list until we either find the value we are looking for:

## Benefits of Binary Search
Binary search is a very efficient way of searching for values in sorted ordinal data.
Binary search is much faster than linear search (except on very small lists).
One drawback of binary search is it can _only_ be used to search ordinal (i.e. sortable) data and that data must be sorted for binary search to work.
However, we will learn later that this might not necessarily be a problem.
